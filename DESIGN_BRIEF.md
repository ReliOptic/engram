# Engram — Product Design Brief

**의뢰인:** Engram 개발팀  
**문서 목적:** UI/UX 리디자인 전문 의뢰  
**작성일:** 2026-04-29  
**긴급도:** High — 첫 팀 배포 전 완성 목표

---

## 1. 제품 이해

### 무엇을 만드는가

Engram은 현장 엔지니어를 위한 **3-에이전트 AI 지원 시스템**이다.

엔지니어가 기술 문제를 텍스트로 입력하면, 3개의 AI 에이전트가 실시간으로 협업한다:
- **Analyzer** — 증상을 분석하고 근본 원인 후보를 확률과 함께 제시
- **Finder** — 유사한 과거 케이스와 매뉴얼을 검색해 증거로 인용
- **Reviewer** — 제안된 해결책이 공식 절차와 일치하는지 검증

대화가 끝나면 해결 내용이 자동으로 지식 베이스에 저장된다.
다음에 같은 문제가 발생하면, 이번 해결 내용이 즉시 인용된다.

### 핵심 사용자 맥락

> "새벽 3시, 반도체 장비 알람. 매뉴얼 찾고 전임자에게 전화하는 대신  
> Engram에 증상을 입력하면 — 30초 안에 비슷한 케이스 3개와 검증된 절차가 나온다."

**사용자 프로필:**
- 반도체·제조·의료기기 등 기술 장비를 다루는 현장 엔지니어
- 전문가이지만 AI에는 낯선 사람들 (UX가 단순해야 한다)
- 시간 압박 상황에서 사용 (빠른 정보 접근이 우아함보다 중요)
- 팀 단위 사용 — 동료의 해결 경험을 공유

**사용 빈도:** 하루 5~15회, 각 세션 5~20분

### 현재 디자인의 강점 (유지할 것)

- **3컬럼 구조** — 에이전트 상태, 대화, 소스를 동시에 보는 것이 핵심 가치
- **에이전트 컬러 구분** — Analyzer(파랑), Finder(라이트블루), Reviewer(회색)
- **기여 타입 배지** — NEW_EVIDENCE, COUNTER, REVISE, PASS의 시각 구분
- **소스 배지 칩** — 텍스트 내 출처를 클릭 가능한 칩으로 분리

### 현재 디자인의 문제점 (해결해야 할 것)

#### 🔴 Critical

**P1. 에이전트가 "무엇을 하고 있는지" 불분명**  
현재: 22px 컬러 사각형 + "thinking" 텍스트  
문제: 실제로 에이전트가 살아있는지, 무언가 진행 중인지 느낌이 없다  
원하는 것: 에이전트가 분석 중일 때 명확한 생동감, 완료 시 결과 요약

**P2. 메시지 버블이 너무 밀도가 높다**  
현재: 모든 버블이 동일한 시각 무게, 벽처럼 쌓임  
문제: COUNTER(중요한 반박)와 NEW_EVIDENCE(일반 근거)가 시각적으로 비슷하다  
원하는 것: 기여의 중요도가 시각적 계층으로 반영됨

**P3. 세션 종료 후 피드백/요약이 어색하다**  
현재: 회색 바 2개가 타임라인 하단에 추가됨 (내부 툴처럼 느껴짐)  
원하는 것: 자연스럽게 대화를 마무리하는 완료 화면

#### 🟡 Medium

**P4. 히스토리 사이드바가 단순 목록이다**  
현재: 제목만 있는 목록, 작은 글씨  
원하는 것: 날짜 그룹, 미리보기 텍스트, 상태 표시

**P5. 지식 베이스 패널이 숫자만 보여준다**  
현재: 컬렉션별 청크 수 테이블  
원하는 것: 지식이 성장하고 있다는 느낌, 최근 추가된 내용

**P6. 설정 페이지가 폼의 나열이다**  
현재: 탭 + HTML 테이블/인풋 조합  
원하는 것: 각 설정의 의미와 영향을 보여주는 설명적 UI

**P7. 다크 모드 없음**  
현장 엔지니어는 종종 어두운 환경에서 작업한다

#### 🟢 Polish

**P8. 빈 화면 (첫 실행) 경험 없음**  
현재: "3 Agents Ready" 텍스트 + 3개 아이콘  
원하는 것: 처음 사용하는 사람을 위한 온보딩 화면

**P9. 모바일/태블릿 레이아웃 없음**  
현장에서 태블릿 사용 가능성 있음

## 3. 디자인 목표

### 목표 1: 신뢰감 있는 전문 도구
"이 AI가 내 판단을 도와준다"는 느낌.  
ChatGPT처럼 가볍지 않고, 엔터프라이즈 소프트웨어처럼 딱딱하지도 않다.  
Linear, Vercel Dashboard, Raycast의 중간 어딘가.

### 목표 2: 에이전트가 살아있음을 느끼게
대화가 진행될 때 에이전트들이 서로 논쟁하고 협력하는 **드라마**를 시각화.  
분석 중 → 반박 → 재검토 → 합의 — 이 흐름이 보여야 한다.

### 목표 3: 지식이 쌓이고 있음을 보여주기
사용할수록 시스템이 똑똑해진다는 핵심 가치를 UI로 전달.  
새 케이스가 추가될 때, 드리밍이 실행될 때 — 무언가 의미 있는 일이 일어나고 있음을 느끼게.

### 목표 4: 극한 상황에서 읽힘
새벽 3시, 압박감 있는 현장. 폰트는 크고, 대비는 높고, 정보는 빠르게 스캔됨.

---

## 4. 디자인 원칙

1. **정보 밀도 > 시각적 아름다움**  
   기능을 숨기지 않는다. 3개 컬럼은 이유가 있어서 있다.

2. **에이전트는 인격이 있다**  
   Analyzer는 분석적, Finder는 탐정 같은, Reviewer는 신중한.  
   각각의 개성이 UI에 반영되어야 한다.

3. **근거가 보여야 신뢰가 생긴다**  
   소스 인용, 확률 표시, 불확실성 표현 — 이것들이 UI의 1급 시민이다.

4. **완성된 케이스가 가장 좋은 마케팅이다**  
   케이스 종료 화면은 가장 만족스러운 순간이어야 한다.

## 5. 화면별 상세 요구사항

### 5-1. 메인 채팅 화면 (가장 중요)

**에이전트 패널 (왼쪽 상단)**
- 현재: 작은 컬러 블록 3개
- 개선: 각 에이전트가 더 큰 카드처럼 표시됨
  - idle: 조용히 대기 중인 느낌
  - thinking: 명확한 활성 상태 (애니메이션, 진행 표시)
  - done: 기여 요약 (예: "5개 케이스 검색 완료")

**채팅 타임라인 (중앙)**
- 기여 타입별 시각 계층:
  - `NEW_EVIDENCE`: 표준 버블
  - `COUNTER`: 주황/레드 왼쪽 보더 + 눈에 띄는 배지 — "이의 제기!"
  - `REVISE`: 노란 왼쪽 보더 — "수정함"
  - `ASK_STAKEHOLDER`: 보라색 — "질문"
  - `PASS`: 최소화된 표현 — 회색, 작게
- 소스 배지: 텍스트에서 분리된 인라인 칩, 클릭하면 우측 패널에 상세 표시
- 에이전트 아바타: 각 버블 좌측에 개성 있는 아이콘

**케이스 완료 화면**
- "케이스가 저장되었습니다" 느낌의 완료 상태
- 합의/이견/패스 통계를 작은 그래프 또는 칩으로 표시
- 피드백 버튼이 자연스럽게 통합됨

**입력 창**
- 사일로 선택 (Account/Tool/Component)이 더 명확하게
- 이미지 첨부 버튼 (현재 API 있음, UI만 필요)
- Send/Stop 버튼이 처리 상태에 따라 명확하게 전환

### 5-2. 히스토리 사이드바 (왼쪽 하단)

- 날짜별 그룹 (오늘, 어제, 이번 주, 이전)
- 각 항목: 제목 + 날짜 + 상태 표시 (open/closed) + 짧은 미리보기
- 현재 세션 강조 표시
- 삭제/이름변경 hover 액션

### 5-3. 지식 베이스 패널 (오른쪽 상단)

- 각 컬렉션을 숫자만이 아닌 의미로 표현
  - "892개 매뉴얼 청크" → "92개 문서, 892개 섹션"
- 마지막 드리밍 실행 시각 + 상태
- "지식이 성장 중" 시각 표현 (최근 추가 수 등)

### 5-4. 소스 사이드바 (오른쪽 하단)

- 소스 목록: 타입 아이콘 + 제목 + 관련도 바
- 청크 상세 패널: 문서 본문이 읽기 좋게 표시됨
- "목록으로" 네비게이션이 명확

### 5-5. 설정 페이지

- 각 탭이 기능을 설명하는 작은 설명 포함
- API Keys 탭: 키 입력 + 연결 테스트 결과가 명확하게 표시
- VectorDB 탭: 드리밍 실행 버튼이 눈에 띄게
- Costs 탭: 총 비용이 크게, 모델별 내역이 테이블로

## 6. 기술 제약사항 (반드시 준수)

| 제약 | 이유 |
|------|------|
| CSS Variables 기반 스타일링 | 기존 컴포넌트 전체가 인라인 스타일 사용 |
| Tailwind 없음 | 프로젝트 미설치, 추가 불가 |
| React 인라인 스타일 또는 CSS Variables | 외부 UI 라이브러리 추가 최소화 |
| Space Grotesk 폰트 유지 | 이미 로드됨 |
| 3-컬럼 레이아웃 유지 | 핵심 정보 구조, 리사이저블 |
| `--brand-primary: #141E8C` | 브랜드 색상 고정 |

```css
--brand-primary: #141E8C    /* 메인 브랜드 색 */

--agent-analyzer: #2563EB   /* 에이전트별 고유 색 */

```

## 9. 프로젝트 컨텍스트

### 코드베이스 접근

디자이너가 원한다면 실제 코드를 공유할 수 있다.
주요 컴포넌트 파일:
- `frontend/src/components/ChatTimeline.tsx` — 메시지 버블 구조
- `frontend/src/components/AgentPanel.tsx` — 에이전트 상태
- `frontend/src/components/ResizableLayout.tsx` — 레이아웃 구조
- `frontend/src/index.css` — 현재 CSS 변수 전체

### 개발 구현 방법

디자이너가 제공한 Figma를 바탕으로 React 인라인 스타일로 직접 구현.  
Figma Dev Mode에서 CSS Values를 추출 가능하도록 작업 바람.

## 10. 일정 및 커뮤니케이션

| 단계 | 내용 |
|------|------|
| 킥오프 | 이 문서 공유 + 질의응답 (30분) |
